Top 10 Common C Programming Errors Examples, Fixes & Explanations

Posted in 10 Computer Science

Introduction

C is a powerful programming language, but beginners often make small mistakes that can cause big problems. Here are 10 common C programming errors, their corrected versions, and explanations so you can avoid them in your own code.

Examples & Fixes

  1. Missing Semicolon 
    // With Error
#include <stdio.h>

int main() {
    int number = 42
    printf("Number: %d\n", number);
    return 0;
}
    // Fixed
#include <stdio.h>

int main() {
    int number = 42; // Added semicolon
    printf("Number: %d\n", number);
    return 0;
}

    Explanation: In C, each statement must end with a semicolon. Without it, the compiler will throw a syntax error.

  2. Missing Semicolon in Multiple Declarations 
    // With Error
#include <stdio.h>

int main() {
    int x = 5, y = 10
    printf("x + y = %d\n", x + y);
    return 0;
}
    // Fixed
#include <stdio.h>

int main() {
    int x = 5, y = 10; // Added semicolon
    printf("x + y = %d\n", x + y);
    return 0;
}

    Explanation: When declaring multiple variables in one line, the declaration must end with a semicolon.

  3. Undeclared Variable 
    // With Error
#include <stdio.h>

int main() {
    int a = 7;
    sum = a + 3;
    printf("Sum = %d\n", sum);
    return 0;
}
    // Fixed
#include <stdio.h>

int main() {
    int a = 7;
    int sum = a + 3; // Declared sum before using it
    printf("Sum = %d\n", sum);
    return 0;
}

    Explanation: All variables must be declared before they are used in C.

  4. Typo in Variable Name 
    // With Error
#include <stdio.h>

int main() {
    int height = 180;
    printf("Height: %d\n", heigth);
    return 0;
}
    // Fixed
#include <stdio.h>

int main() {
    int height = 180;
    printf("Height: %d\n", height); // Corrected spelling
    return 0;
}

    Explanation: Misspelled variable names cause compile-time errors because the compiler cannot find that identifier.

  5. Wrong Format Specifier for Float 
    // With Error
#include <stdio.h>

int main() {
    float price = 19.99;
    printf("Price: %d\n", price);
    return 0;
}
    // Fixed
#include <stdio.h>

int main() {
    float price = 19.99;
    printf("Price: %.2f\n", price); // Changed %d to %.2f
    return 0;
}

    Explanation: %d is for integers; %f or %.2f is for floating-point numbers.

  6. Wrong Format Specifier for Integer 
    // With Error
#include <stdio.h>

int main() {
    int age = 25;
    printf("Age: %f\n", age);
    return 0;
}
    // Fixed
#include <stdio.h>

int main() {
    int age = 25;
    printf("Age: %d\n", age); // Changed %f to %d
    return 0;
}

    Explanation: %f is for floats/doubles; %d is for integers.

  7. Missing Closing Quote in String 
    // With Error
#include <stdio.h>

int main() {
    printf("Hello, World!\n);
    return 0;
}
    // Fixed
#include <stdio.h>

int main() {
    printf("Hello, World!\n"); // Added closing quote
    return 0;
}

    Explanation: Strings in C must start and end with double quotes.

  8. Mismatched Braces 
    // With Error
#include <stdio.h>

int main() {
    int a = 10;
    if (a > 5) {
        printf("a is greater than 5\n");
    // Missing closing brace
}
    // Fixed
#include <stdio.h>

int main() {
    int a = 10;
    if (a > 5) {
        printf("a is greater than 5\n");
    } // Closed if block
} // Closed main function

    Explanation: Every opening brace { must have a matching closing }.

  9. Extra Comma in Variable Declaration 
    // With Error
#include <stdio.h>

int main() {
    int a, b,;
    a = 2;
    b = 3;
    printf("Sum = %d\n", a + b);
    return 0;
}
    // Fixed
#include <stdio.h>

int main() {
    int a, b; // Removed extra comma
    a = 2;
    b = 3;
    printf("Sum = %d\n", a + b);
    return 0;
}

    Explanation: Extra commas are not allowed in variable declarations.

  10. Wrong Return Type for main 
    // With Error
#include <stdio.h>

void main() {
    printf("Program Running...\n");
    return 0;
}
    // Fixed
#include <stdio.h>

int main() { // Changed void to int
    printf("Program Running...\n");
    return 0; // Proper return value for int main
}

    Explanation: The standard return type for main is int, and it should return 0 on success.

Conclusion

Most C programming errors are small syntax issues or misunderstandings about the language. By learning to spot these mistakes early, you can save time debugging and write cleaner, more reliable code.
